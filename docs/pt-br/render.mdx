# # Compreendendo o Plano

Render Blueprints são arquivos YAML que definem sua infraestrutura. O <<CODE0> neste
repositório configura tudo o que é necessário para executar OpenClaw:

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: PORT
        value: "8080"
      - key: SETUP_PASSWORD
        sync: false # prompts during deploy
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # auto-generates a secure token
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

# # Render Dashboard recursos

Diários

Ver logs em tempo real em **Dashboard → seu serviço → Logs**. Filtro por:

- Registros de compilação (criação de imagem Docker)
- Implantar logs (início do serviço)
- Registros de execução (saída de aplicação)

Acesso da Shell

Para depuração, abra uma sessão shell via **Dashboard → seu serviço → Shell**. O disco persistente é montado em <<CODE0>>>.

## # Variáveis de ambiente

Modificar variáveis em **Dashboard → seu serviço → Ambiente**. As alterações desencadeiam uma transferência automática.

Auto-colar

Se você usar o repositório OpenClaw original, o Render não enviará automaticamente seu OpenClaw. Para atualizá-lo, execute uma sincronização manual Blueprint do painel.

# # Domínio personalizado

1. Vá para **Dashboard → seu serviço → Configurações → Domínios personalizados**
2. Adicione seu domínio
3. Configurar DNS conforme instruído (CNAME para <<CODE0>>)
4. Render disposições um certificado TLS automaticamente

# # Escala

Render suporta escala horizontal e vertical:

- **Vertical**: Alterar o plano para obter mais CPU/RAM
- **Horizontal**: Aumentar a contagem de instâncias (plano padrão e superior)

Para OpenClaw, a escala vertical geralmente é suficiente. A escala horizontal requer sessões fixas ou gerenciamento externo do estado.

# # Backups e migração

Exportar sua configuração e espaço de trabalho a qualquer momento:

```
https://<your-service>.onrender.com/setup/export
```

Isso baixa um backup portátil que você pode restaurar em qualquer host OpenClaw.

# # Resolução de problemas

O serviço não vai começar

Verifique os registos de implantação no Painel Render. Questões comuns:

- Faltando <<CODE0>> — o Blueprint pede para isso, mas verifique se está definido
- Correspondência de porta — garantir que <<CODE1> corresponda à porta exposta do Dockerfile

Começa o frio lento

Serviços de nível livre giram para baixo após 15 minutos de inatividade. O primeiro pedido após a rotação leva alguns segundos enquanto o recipiente começa. Atualizar para o plano inicial para sempre.

Perda de dados após reafectação

Isso acontece no nível livre (sem disco persistente). Atualização para um plano pago, ou
exportar regularmente a sua configuração via <<CODE0>>>.

Falhas na verificação de saúde

Render espera uma resposta de 200 <<CODE0>> em 30 segundos. Se a compilação tiver sucesso, mas a implantação falhar, o serviço pode demorar muito para começar. Verificar:

- Criar logs para erros
- Se o recipiente é executado localmente com <<CODE0>
